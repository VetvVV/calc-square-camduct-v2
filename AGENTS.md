## 4. Git-режим

Безопасный процесс по умолчанию:

1. branch;
2. commit;
3. PR;
4. merge;
5. pull main;
6. test / build / lint;
7. cleanup.

Если владелец явно просит работать локально в `main`:

- можно делать commit;
- нельзя делать push без отдельной команды владельца;
- нельзя делать deploy без отдельной команды владельца;
- нельзя запускать `PUBLISH.bat`;
- нельзя делать force push;
- нельзя удалять рабочие файлы без явной необходимости.

Перед изменениями всегда проверить:

```bash
git status --short --branch
git log --oneline -5
```

После изменений выполнить релевантные проверки:

```bash
npm.cmd test -- --run <релевантные тесты>
npm.cmd run build
npm.cmd run lint
```

После `build` обязательно проверить `src/build-info.ts`.

Если `build` изменил `src/build-info.ts`, восстановить его перед commit:

```bash
git restore --worktree -- src/build-info.ts
```

В финальном отчёте показать:

```bash
git diff --stat HEAD~1..HEAD
git log --oneline -3
git status --short --branch
```

## 9. Единицы измерения

Все вводимые размеры — в миллиметрах.

Площадь выводится в квадратных метрах.

Количество умножает итоговую площадь.

Толщина — в миллиметрах.

Масса, если считается, должна быть отдельной функцией.

Не смешивать:

- площадь;
- массу;
- спецификацию;
- connector / BOM logic.

## 10. Публичные обозначения и CAMduct mapping

В публичном UI нельзя показывать CAMduct `A/B/C/D` как основные размерные обозначения.

Публичный UI использует понятные инженерные обозначения:

```text
D — диаметр
L — длина
W — ширина
H — высота
```

CAMduct mapping хранится отдельно от публичного UI.

Mapping индивидуален для каждого изделия.

Нельзя глобально считать, что CAMduct `A` всегда ширина или всегда диаметр.

Пример:

```text
CAMduct A → публичный D
CAMduct B → публичный L
```

Для разных изделий mapping может быть другим.

## 13. R-001 — труба прямошовная

Изделие:

```text
ID: R-001
Название: Труба прямошовная
Категория: Круглые
```

Публичные поля:

```text
D — диаметр, мм
L — длина, мм
Количество
Материал
Толщина
Опции
Комментарий
```

CAMduct mapping для R-001:

```text
CAMduct A → D / Диаметр
CAMduct B → L / Длина
CAMduct C → левое удлинение, disabled / не используется
CAMduct D → правое удлинение, disabled / не используется
```

Публичное правило:

- показывать `D` и `L`;
- не показывать CAMduct `A/B/C/D`;
- не использовать `ØD` без отдельного подтверждения;
- рисунок должен использовать только `D` и `L`.

Материалы R-001:

```text
Оцинкованная сталь
Нержавеющая 430 техническая
Нержавеющая 304 пищевая
```

Не добавлять без подтверждения:

```text
Алюминий
AISI 316
AISI 321
другие материалы
```

Минимальная подтверждённая формула площади R-001:

```text
S = π × D × L / 1_000_000
S_total = S × qty
```

Проверочный пример:

```text
D = 250 мм
L = 1000 мм
S ≈ 0.785 м²

qty = 2
S_total ≈ 1.571 м²
```

## 16. Правила толщины

Круглые изделия: толщина по диаметру `D`.

```text
D <= 499          → 0.5 мм
500 <= D <= 999  → 0.7 мм
D >= 1000        → 0.9 мм
```

Прямоугольные изделия: толщина по максимальной стороне `max(W, H)`.

```text
max(W, H) <= 499          → 0.5 мм
500 <= max(W, H) <= 999   → 0.7 мм
max(W, H) >= 1000         → 0.9 мм
```

Не добавлять новые диапазоны толщин без подтверждения.

## 18. S1 / продольный замок

S1 — это производственный продольный замок / шов внутри изделия.

S1 не является торцевым соединителем.

Известная логика:

```text
Американский замок — в основном для прямоугольных изделий.
Русский замок — в основном для круглых изделий с длиной около/выше 450 мм,
а также для больших деталей, которые делятся на части.
```

Русский замок по толщине:

```text
0.5 и 0.7 мм → 12.5 + 12.5 = 25 мм
0.9 мм       → 14 + 14 = 28 мм
```

Не округлять 12.5 до 12 при расчёте полной площади развёртки.

Не добавлять припуски S1 в формулы площади, пока правило не подтверждено для конкретного изделия.

Сохранять корпоративную оболочку текущей версии, если она уже удачна.

Для R-001:

- страница должна выглядеть как рабочий калькулятор, не как демонстрационная карточка;
- watermark `ОПЫТНЫЙ ОБРАЗЕЦ` допустим как полупрозрачный overlay;
- watermark не должен занимать высоту в layout;
- `pointer-events: none`;
- не должен мешать кликам;
- рисунок трубы должен быть крупным, но без лишней пустоты;
- стенка / torus визуально тоньше;
- не показывать отверстия, когда их нет;
- основной CTA — оранжевый;
- не показывать массу в публичном режиме;
- масса — только инженерный / Admin + ⚙ ON.

Если задача не завершена:


Не писать длинный английский отчёт.

Не добавлять лишнюю теорию.

## 27. Текущее состояние, которое важно помнить

Актуальные детали могут меняться, поэтому перед задачей всегда смотреть `git status`.

Из недавней работы известно:

- R-001 diagram есть и расположен рядом с формой;
- роли и badge доступа разделены;
- R-001 language control сделан статичным `RU`;
- отверстия R-001 редактируются в UI;
- отверстия отображаются на рисунке;
- отверстия пока не подтверждены как вычитаемые из площади;
- локальный `main` может быть ahead of origin;
- push / deploy / `PUBLISH.bat` не выполнять без команды владельца.

Не придумывать.

Не спешить.

Не делать «заодно».
